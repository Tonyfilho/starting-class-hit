Programação Orientada a Objetos (OOP) em TypeScript

1. O que é OOP?

A Programação Orientada a Objetos é um paradigma que organiza o código em torno de classes e objetos, permitindo reutilização, abstração e encapsulamento.

  • Classe: Molde para criar objetos com propriedades e métodos.
  • Objeto: Instância de uma classe.
  • Encapsulamento: Oculta detalhes internos, expondo apenas o necessário.
  • Herança: Permite que uma classe herde de outra.
  • Polimorfismo: Métodos com mesmo nome se comportam de forma diferente em classes distintas.

4. Modificadores de Acesso

  • public: Acesso livre (padrão).
  • private: Acessível apenas dentro da classe.
  • protected: Acessível na classe e em subclasses.
  • readonly: Propriedade que só pode ser lida (não modificada após atribuída).
class Person {{"{"}}
  public name: string;
  private age: number;
  protected role: string;
  readonly id: number;

  constructor(name: string, age: number, role: string, id: number) {{"{"}}
    this.name = name;
    this.age = age;
    this.role = role;
    this.id = id;
  {{"}"}}

  getAge(): number {{"{"}}

Resumo Técnico

  • ✅ Classes definem estruturas e comportamentos de objetos.
  • ✅ A herança permite reutilizar código entre classes relacionadas.
  • ✅ Modificadores controlam a visibilidade de membros.
  • ✅ Getters e setters encapsulam o acesso a propriedades.
  • ✅ Classes abstratas definem contratos para subclasses.
  • ✅ Interfaces descrevem a forma dos objetos sem implementação.
