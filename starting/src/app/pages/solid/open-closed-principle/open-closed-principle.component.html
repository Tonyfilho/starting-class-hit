Open/Closed Principle (OCP)

O que é o Open/Closed Principle?

O OCP afirma que as classes devem estar abertas para extensão, mas fechadas para modificação.
Isso significa que devemos poder adicionar novos comportamentos ao código sem alterar seu funcionamento interno.

Exemplo violando o OCP

Classe alterada a cada novo tipo


    class DescontoService {
        calcular(tipo) {
            if (tipo === 'cliente') return 10;
            if (tipo === 'vip') return 20;
            if (tipo === 'premium') return 30;
        }
    }

Qualquer novo tipo exige modificação direta da classe.

Aplicando o OCP

Uso de polimorfismo e extensão


    interface Desconto {
    getValor(): number;
    }

    class DescontoVip implements Desconto {
    getValor() { return 20; }
    }

    class DescontoPremium implements Desconto {
    getValor() { return 30; }
    }

Agora, adicionamos novos tipos sem modificar os anteriores.

Resumo Técnico

  • ✅ O código é mais estável e flexível.
  • ✅ Evita alterações em código já testado.
  • ✅ Favorece a extensão através de herança ou interfaces.
