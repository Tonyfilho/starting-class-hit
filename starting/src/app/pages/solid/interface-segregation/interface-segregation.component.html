Interface Segregation Principle (ISP)

O que é o Interface Segregation Principle?

O ISP afirma que classe deve ser forçada a depender de métodos que não utiliza.
Interfaces devem ser específicas para seus clientes.

Exemplo violando o ISP

Interface genérica demais


     interface Funcionario {
        trabalhar(): void;
        gerenciar(): void;
    }

    class Desenvolvedor implements Funcionario {
        trabalhar() { ... }
        gerenciar() { throw "Não gerencio"; }
    }

O desenvolvedor é forçado a implementar métodos que não fazem sentido para ele.

Aplicando o ISP

Interfaces separadas por função


    interface Trabalhador { trabalhar(): void; }
    interface Gerente { gerenciar(): void; }

    class Desenvolvedor implements Trabalhador {
        trabalhar() { ... }
    }

    class Coordenador implements Trabalhador, Gerente {
        trabalhar() { ... }
        gerenciar() { ... }
    }

Resumo Técnico

  • ✅ Interfaces específicas evitam implementações inúteis.
  • ✅ Cada classe implementa apenas o que de fato usa.
  • ✅ Favorece código limpo e coeso.
