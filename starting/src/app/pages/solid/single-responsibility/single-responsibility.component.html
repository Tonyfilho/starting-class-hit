Single Responsibility Principle (SRP)

O que é o Princípio da Responsabilidade Única?

O SRP afirma que classe deve ter um, e apenas um, motivo para mudar. Em outras palavras, ela deve ser responsável por apenas uma parte do funcionamento do sistema.
Isso facilita a manutenção, os testes, e a reutilização do código.

Exemplo com Violação do SRP

Veja esta classe que faz mais de uma coisa:

Classe com múltiplas responsabilidades


    class User {
        saveUserToDatabase() { ... }
        validateEmail() { ... }
        sendWelcomeEmail() { ... }
    }

Essa classe tem 3 responsabilidades: salvar no banco, validar email, e enviar email.

Aplicando o SRP

Agora veja como podemos separar essas responsabilidades:

Separação em classes especializadas


    class UserRepository {
        save(user) {  ... }
    }

    class EmailValidator {
        isValid(email) {  ... }
    }

    class EmailService {
        sendWelcomeEmail(user) {  ... }
    }

Agora cada classe tem uma responsabilidade única e clara.

Resumo Técnico

  • ✅ SRP melhora a organização e a clareza do código.
  • ✅ Cada classe deve ter um motivo único para mudar.
  • ✅ Facilita manutenção, testes e reuso.
