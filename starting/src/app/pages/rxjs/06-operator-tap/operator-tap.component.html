Operador Tap no RxJS

1. O que é o tap?

O operador tap do RxJS permite executar efeitos colaterais para cada valor emitido pelo Observable, sem alterar o valor em si. Ele é muito usado para debug, logging ou chamadas externas que não devem afetar o fluxo.

  • 🔹 É semelhante ao console.log dentro de um pipe().
  • 🔹 Não modifica os dados emitidos.
  • 🔹 Ideal para depuração ou ações pontuais.

2. Exemplo Prático

Vamos usar o tap para fazer um log antes de aplicar uma transformação com map:


import {{"{"}} of {{"}"}} from 'rxjs';
import {{"{"}} tap, map {{"}"}} from 'rxjs/operators';

of(1, 2, 3)
  .pipe(
    tap(valor => console.log('Antes do map:', valor)),
    map(valor => valor * 10),
    tap(valor => console.log('Depois do map:', valor))
  )
  .subscribe(resultado => console.log('Resultado:', resultado));

// Saída:
// Antes do map: 1
// Depois do map: 10
// Resultado: 10
// ...

3. Quando Usar?

  • ✅ Para fins de debug/logging durante o desenvolvimento.
  • ✅ Para disparar ações auxiliares (ex: mostrar um loading, evento de analytics).
  • ✅ Sempre que precisar “ver” o valor, sem transformá-lo.

4. Com Operações Assíncronas

O tap é útil antes e depois de chamadas assíncronas para controle e rastreamento: 


tap(() => this.loading = true),
switchMap(() => this.apiService.getData()),
tap(() => this.loading = false)

Resumo Técnico

  • ✅ tap executa efeitos colaterais sem alterar o valor emitido.
  • ✅ Muito útil para debug, rastreamento e ações auxiliares.
  • ✅ Não deve ser usado para alterar o fluxo de dados.
