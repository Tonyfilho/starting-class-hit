Directives

Em Angular, as diretivas são classes que adicionam comportamentos personalizados aos elementos HTML ou modificam sua estrutura no DOM. Elas são uma parte fundamental do Angular, permitindo estender a funcionalidade dos templates de forma declarativa.

No Angular, as diretivas {{ "@if, @for e ngClass" }} são recursos poderosos para controlar a renderização condicional, repetição de elementos e estilização dinâmica em templates. Elas substituem ou aprimoram funcionalidades que antes dependiam do Angular tradicional (como *ngIf e *ngFor), trazendo uma sintaxe mais intuitiva e eficiente.

{{"@if(){...}"}}

A diretiva {{ "@if" }} permite exibir um bloco de conteúdo somente se uma condição for verdadeira. Diferente do *ngIf (que era estrutural), o {{ "@if" }} usa uma sintaxe mais limpa e direta, sem necessidade de asterisco (*). Ele também suporta blocos {{"@else"}} e {{"@else if " }} para condições alternativas, tornando a lógica mais legível. Principais características: Avalia uma expressão booleana para decidir se o conteúdo é renderizado. Remove o elemento do DOM quando a condição é falsa (otimização de desempenho). Pode ser combinado com {{"@else"}} para casos alternativos.

@if (this.showMessage) {

}

{{"@for(){...}"}}

{{ "@for" }} – Repetição de Elementos A diretiva {{ "@for" }} é usada para iterar sobre uma lista e renderizar um bloco de template para cada item. Ela substitui o *ngFor, oferecendo melhor desempenho e sintaxe simplificada. Principais características: Gera elementos dinamicamente com base em um array ou coleção. Oferece variáveis de contexto como $index (índice atual), $first, $last, etc. Suporta {{" @empty" }} para exibir um bloco alternativo quando a lista está vazia.

@for (item of this.items; track $index) {

Mostra a Mensagem do For {{ item + ' Elevador Andar: '}} {{ $index }}

}

{{"ngClass{...}"}}

ngClass Classes CSS Dinâmicas O ngClass é uma diretiva de atributo que aplica classes CSS condicionalmente a um elemento. Ele permite adicionar ou remover classes com base em expressões, objetos ou arrays, tornando a estilização reativa e flexível. Principais características: Aceita um objeto onde as chaves são nomes de classes e os valores são condições booleanas. Pode receber um array de strings (classes fixas) ou uma string única. Ideal para alternar estilos com base no estado da aplicação (ex: ativo/desativado, temas).
