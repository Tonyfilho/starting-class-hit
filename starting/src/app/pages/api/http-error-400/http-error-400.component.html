Erro 400 – Requisição Inválida

A requisição enviada pelo cliente tem algum problema. Normalmente é causado por erros de sintaxe, parâmetros inválidos ou falta de autenticação.

O que significa esse erro?

  • 400 Bad Request: Requisição malformada (ex: JSON inválido, parâmetros errados).
  • 401 Unauthorized: Requisição sem autenticação ou com token inválido.
  • 403 Forbidden: Autenticado, mas sem permissão de acesso.
  • 404 Not Found: Recurso não encontrado (URL errada, por exemplo).
  • 405 Method Not Allowed: Método HTTP inválido para esse endpoint.
  • 409 Conflict: Conflito de estado, como dados duplicados.
  • 422 Unprocessable Entity: Dados válidos na sintaxe, mas inválidos na lógica.
  • 429 Too Many Requests: Limite de requisições excedido.

Principais Códigos da Classe 400

Revise a requisição no cliente (formulários, JSON, headers, etc.). Certifique-se de que os dados estejam corretos e completos
