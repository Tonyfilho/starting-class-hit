Ferramentas Utilizadas

Angular

Framework de desenvolvimento web baseado em TypeScript, mantido pelo Google. É usado para criar aplicações web de página única (SPAs) com uma arquitetura robusta e escalável.

Git Extensions

Interface gráfica para o Git no Windows. Facilita o gerenciamento de repositórios Git com recursos visuais para commits, branches, merges e muito mais, ideal para quem prefere evitar o terminal.

Cmder

Emulador de terminal para Windows que oferece uma interface mais poderosa e amigável do que o Prompt padrão. Baseado no ConEmu, integra comandos Unix via clink e Git para Windows.

Príncipios da Programação

DRY

Princípio que incentiva a evitar duplicação de código. A ideia é que cada pedaço de conhecimento ou lógica deve existir em um único lugar do sistema, facilitando a manutenção e reduzindo erros.

SOLID

Conjunto de cinco princípios para um design orientado a objetos mais compreensível, flexível e fácil de manter:
S – Single Responsibility Principle:
O – Open/Closed Principle:
L – Liskov Substitution Principle:
I – Interface Segregation Principle:
D – Dependency Inversion Principle:

Object Calisthenics

Conjunto de 9 regras que visam melhorar a qualidade do código orientado a objetos por meio de restrições que forçam boas práticas.

Nivelamentos

Angular

Consiste em componentes, módulos, diretivas, serviços, injeção de dependência, roteamento e consumo de APIs.

JavaScript

Inclui variáveis, funções, objetos, arrays, escopo, promises, async/await e manipulação do DOM. Essencial para programar com segurança em frameworks modernos como Angular.

RxJS

Envolve a introdução aos conceitos de programação reativa com a biblioteca RxJS, usada extensivamente no Angular. Inclui entender Observables, operadores (como map, filter, switchMap), Subjects e o padrão de fluxo assíncrono e reativo.

Angular Foundation

É a base de conhecimentos essenciais para começar a trabalhar com Angular de forma sólida. Envolve os principais conceitos e estruturas do framework, como: Componentes e Templates, Módulos (NgModules), Data Binding, Diretivas, Serviços e Injeção de Dependência, Roteamento (Router), HTTPClient

Firebase Init

Comando (firebase init) usado para iniciar e configurar um projeto Firebase no seu ambiente local. Durante a execução, você escolhe quais serviços deseja ativar (como Hosting, Firestore, Functions etc.). Ele cria os arquivos de configuração necessários, como firebase.json e .firebaserc.

Firebase Hosting

Serviço do Firebase para hospedar sites estáticos ou SPAs (como projetos em Angular, React, Vue). É rápido, seguro (HTTPS por padrão) e fácil de usar. Após o firebase init, você pode usar firebase deploy para publicar seu site online.

HTTP Errors e Successes

HTTP Error - 400

Bad Request: a requisição foi malformada ou contém dados inválidos (ex: parâmetros faltando).

HTTP Error - 500

Internal Server Error: problema no servidor ao processar a requisição, geralmente causado por bugs ou falhas internas.

HTTP Success - 200

OK: a requisição foi bem-sucedida e o servidor retornou a resposta esperada (ex: dados de uma API ou confirmação de operação).

APIs

CORS

Mecanismo de segurança que controla quais domínios podem acessar recursos de uma API hospedada em outro domínio. sem CORS habilitado, requisições entre origens diferentes (ex: de localhost para api.exemplo.com) são bloqueadas pelo navegador por segurança.

Weather API

API que fornece dados meteorológicos em tempo real, como temperatura, umidade e previsão do tempo.

Reqres API

API gratuita usada para testes e simulações de requisições HTTP (GET, POST, PUT, DELETE). Retorna dados fictícios como usuários, listas e autenticação. Muito útil em protótipos, testes e aprendizado de consumo de APIs.
